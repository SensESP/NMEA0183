# NMEA0183 Release Process

Based on the SensESP release process with simplifications (no autogenerated docs step).

## Prerequisites

- PlatformIO CLI (`pio`)
- bump2version (`pip install bump2version`)
- PlatformIO registry permissions for the `sensesp` owner

## Process

1. Create a release branch `release_x.y.z` from `main`.

2. Bump the version with bumpversion.

   Development versions are `x.y.z-alpha`. To release:
   - `bumpversion release` — bumps `x.y.z-alpha` → `x.y.z`
   - `bumpversion minor` then `bumpversion release` — bumps to next minor
   - `bumpversion major` then `bumpversion release` — bumps to next major
   - `bumpversion --new-version x.y.z <part>` — set a specific version

   Bumpversion creates a commit automatically.

3. Push the branch, create a PR against `main`, and merge it.

4. Create a GitHub release at https://github.com/SensESP/NMEA0183/releases.
   Tag: `vx.y.z`. Title: `Version x.y.z`. Include a summary of changes.

5. Publish to the PlatformIO registry:

       pio package publish --owner sensesp --no-interactive

   The publish command packages your current worktree, so ensure it is clean.

6. Post-release version bump. Create a branch `post_release_x.y.z`, then:

       bumpversion patch

   This bumps to `x.y.(z+1)-alpha`. Push, create a PR, and merge.
